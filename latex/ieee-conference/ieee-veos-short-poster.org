\begin{block}{Problem}
\justifying
Electric vehicles operate in noisy, partially observed, and non-repeatable road environments. Drivers can consume different energy on the same route because acceleration and braking dominate short-horizon energy demand.

\vspace{0.6em}
\begin{itemize}
\item Simulation-only eco-driving often assumes connected traffic, perfect preview, or autonomous speed control.
\item Shaped rewards and soft constraints are fragile under route, weather, and driving-style drift.
\item Production deployment needs bounded actions, fallback behavior, and compatibility with the vehicle control unit.
\end{itemize}
\end{block}

\begin{block}{EOSEV Design Principle}
\begin{itemize}
\item Learn from directly measurable vehicle signals.
\item Use measured electric work as the reward.
\item Adapt the VCU pedal map through bounded torque increments.
\item Preserve driver-in-the-loop production control.
\end{itemize}
\end{block}

\begin{block}{MDP Formulation}
\[
s_t=(V_t,A_t,B_t), \qquad a_t=\Delta\tau_t
\]
\[
r_t=-U_t I_t T_s
\]
\begin{itemize}
\item \(V\): vehicle speed
\item \(A,B\): accelerator and brake openings
\item \(U,I\): battery voltage and current
\item \(\Delta\tau\): bounded torque-request update
\end{itemize}
\end{block}
\end{column}

\begin{block}{Learning and Deployment}
\begin{itemize}
\item Customized DDPG actor-critic training.
\item Replay buffer stores \((s_t,a_t,r_t,s'_t)\) time-series records.
\item Actor-only inference is suitable for a small TFLite deployment.
\item Cloud training can distribute validated model and map updates by OTA.
\end{itemize}

\begin{block}{Proving-Ground Training}
\begin{itemize}
\item Fixed vehicle, driver, route, and speed profile.
\item About 60 episodes per three-hour run.
\item Previous model weights and pedal map are reloaded for continuous training.
\item Driving-style drift is monitored using KL divergence from baseline pedal-opening distribution.
\end{itemize}

\begin{block}{Transferability and Multimodality}
\begin{itemize}
\item Frozen route-1 model reduces consumption by about \(4\%\) on a second route without fine-tuning.
\item Traffic-light and pedestrian halts are handled as multimodal road scenarios.
\item Shown modes reduce energy by \(9.5\%\) without halts and \(4.3\%\) with halts at all turns.
\end{itemize}

\begin{block}{Takeaways}
\begin{itemize}
\item Real-world DRL can be sample efficient when state, action, reward, and deployment are tightly scoped.
\item Bounded pedal-map updates provide a practical automotive action interface.
\item Scaling requires drift monitoring, frozen-transfer validation, and fleet-data learning.
\end{itemize}
